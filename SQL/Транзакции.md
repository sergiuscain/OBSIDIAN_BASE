Транзакции - это механизм, обеспечивающий целостное выполнение программы. (Ряда операций).
Для начала транзакции используем команду "START TRANSACTION"
Для фиксации изменений, сделанных во время транзакции, используем команду "COMMIT".
Либо мы можем не фиксировать изменения, а отменить их. 
Для этого используем команду "COMMIT"
##  Транзакции
```sql
START TRANSACTION;
UPDATE accounts SET balance = balance - 100 WHERE id = 1;
UPDATE accounts SET balance = balance + 100 WHERE id = 2;
COMMIT;
```

Пример транзакции с условной конструкцией 
```sql
DELIMITER //
-- Создание хранимой процедуры для перевода средств между счетами 
CREATE PROCEDURE transfer_funds( IN from_account_name VARCHAR(100), 
-- Имя счета отправителя 
IN to_account_name VARCHAR(100), 
-- Имя счета получателя 
IN amount DECIMAL(10, 2) -- Сумма перевода 
) BEGIN DECLARE from_balance DECIMAL(10, 2); 
-- Переменная для хранения баланса счета отправителя 
DECLARE to_balance DECIMAL(10, 2); 
-- Переменная для хранения баланса счета получателя 
-- Начало транзакции 
START TRANSACTION; 
-- Получаем текущие балансы счетов 
SELECT balance INTO from_balance FROM accounts WHERE account_name = from_account_name; SELECT balance INTO to_balance FROM accounts WHERE account_name = to_account_name; 
-- Проверяем, достаточно ли средств на счете отправителя 
IF from_balance < amount THEN 
-- Если средств недостаточно, откатываем транзакцию 
ROLLBACK; SELECT 'Transaction rolled back due to insufficient funds' AS message; 
ELSE 
-- Если средств достаточно, выполняем перевод средств 
UPDATE accounts SET balance = balance - amount WHERE account_name = from_account_name; UPDATE accounts SET balance = balance + amount WHERE account_name = to_account_name;
-- Фиксируем транзакцию 
COMMIT; SELECT 'Transaction committed successfully' AS message; 
END IF; 
END // DELIMITER ;
```
