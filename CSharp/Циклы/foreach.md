# Описание

Цикл `foreach` используется для итерации по всем элементам коллекции или массива. Он автоматически проходит через все элементы, не требуя управления индексом.
# Особенности foreach

- Нельзя изменять коллекцию во время итерации (вызовет исключение)
- Только для чтения элементов
- Автоматическое приведение типа через `var`
- Простой и безопасный синтаксис
# Синтаксис
Вначале мы указываем тип и имя переменной, которая принимает каждое значение из коллекции, потом, после 'in', мы указываем коллекцию, по которой проходимся.
```csharp
string[] fruits = { "яблоко", "банан", "апельсин" };

foreach (string fruit in fruits)
{
    Console.WriteLine(fruit);
}
// Вывод:
// яблоко
// банан
// апельсин
```
Мы перебираем каждый фрукт в коллекции фрукты.

## пример: Работа со словарем
Здесь каждый элемент коллекции является парой ключ-значение.
Поэтому у каждого элемента есть свойства Key и Value.
В случае, если мы будем работать с собственным типом, мы сможем обращаться так-же к каждому свойству и методу нашего типа.
```csharp
Dictionary<string, int> ages = new Dictionary<string, int>
{
    ["Анна"] = 25,
    ["Иван"] = 30,
    ["Мария"] = 28
};

foreach (var person in ages)
{
    Console.WriteLine($"{person.Key}: {person.Value} лет");
}
```

# С какими коллекциями работает foreach

| Тип коллекции                                                  | Пример                       | Особенность                |
| -------------------------------------------------------------- | ---------------------------- | -------------------------- |
| **Массивы**                                                    | `int[] arr = {1, 2, 3};`     | Любые одномерные массивы   |
| **Списки**                                                     | `List<T>`, `ArrayList`       | Динамические коллекции     |
| **Словари**                                                    | `Dictionary<TKey, TValue>`   | Пары ключ-значение         |
| **Множества**                                                  | `HashSet<T>`, `SortedSet<T>` | Уникальные элементы        |
| **Очереди/Стеки**                                              | `Queue<T>`, `Stack<T>`       | FIFO/LIFO коллекции        |
| **Строки**                                                     | `string`                     | По символам                |
| **Любые типы, реализующие** `IEnumerable` или `IEnumerable<T>` |                              | Пользовательские коллекции |
*Там, где буква T, мы можем подставить свой тип данных.
Например, `List<Свинка>`
И наш тип данных:
public class Свинка
{
	public string Name {get; set;}
	public void Хрюкнуть;
}
Это называется Дженерик коллекция (# Generic collections). 
Так же говорят "Обобщённая коллекция" 
То есть она может принимать любой тип данных.
