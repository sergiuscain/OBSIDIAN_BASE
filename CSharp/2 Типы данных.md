Типы данных в c# делятся на значимые и ссылочные
## -Значимые
Хранят сами данные.
К ним относятся примитивные типы: int, double, bool, char
А так же все структуры (struct) и перечисления (enum)

При передачи значимого типа данных в метод, передается его копия. При её изменении, оригинал не изменяется (вне метода который)

Значимые типы хранятся в стэке.
Исключения: 
- поля классов, 
- упакованные в object переменные (boxing. object - ссылочный тип данных)
- Находится внутри коллекции
## -Ссылочные
Хранят ссылку на объект в куче (адрес места в памяти)
К ним относятся: классы, интерфейсы, делегаты, массивы, строки.

При изменении значения ссылочной переменой в методе, изменяется значение в куче. При это ссылка в методе и вне метода ведёт на одну и туже область памяти и обе переменные будут возвращать новое значение переменной.

НО если внутри метода переменной присвоить значение `= new objectName()`, то переменная внутри метода будет ссылаться на только что созданный объект и никак не будет связанна с переменой снаружи метода.

## -Равенство обектов
Два значимых объекта равны, если равны их значения.
Два ссылочных объекта равны, если равны их ссылки.
НО два типа данных типа string равны, ели равны их значения!! 
(string - ссылочный тип данных, но у него особое поведение)
```c#
int a = 5;
int b = 5;
// a равно b так ка их значения равны

var person1 = new Person(Name = "John"); // Создаем персонажа с именем John
var person2 = new Person(Name = "John"); // Создаем НОВОГО персонажа и именем John
// person1 и person2 не равны так как у них ссылки на разные области памяти, 
// которые были выделенны во время инициализации переменных через 'new'
```

## -Время жизни
- Значимые типы данных удаляются при выходе из области видимости (например, при выходе из переменной)
- Ссылочные типы данных хранятся в кучу и существуют, пока на них ведёт хотя-бы одна ссылка. Когда сборщик мусора проходится и видит объекты в куче без ссылок, он освобождает память от них.
