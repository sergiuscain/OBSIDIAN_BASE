## -Интерфейс
Интерфейс - это контракт, описывающий, что должен делать. Но он не описывает, как он должен это делать. 
В интерфейсе указывается список методов, свойств и событий. А реализуются они в классе, который наследуется от интерфейса.

Класс может реализовывать множество интерфейсов. (а наследование может быть ТОЛЬКО од одного класса!)

Начиная с C# 8.0, в интерфейсах появилась возможность добавить реализацию методов по умолчанию.

Интерфейсы не могут содержать поля.

## -Абстрактные классы
Абстрактный класс - это класс, который не может быть создан напрямую и может содержать как абстрактные методы(без реализации), так и конкретные.

Абстрактные методы нужно обязательно реализовать. 
Конкретные методы будут наследоваться автоматически.

Класс может наследоваться только от одного абстрактного класса.

## -Когда что использовать
### -Интерфейс используется для
- Определение контрактов для не связных классов (Круг, кнопка, текст могут использовать некий интерфейс для отрисовки графики).
- Множественная реализация поведения (класс может иметь множество ролей одновременно. 
```C#
class SmartDevice : IConnectable, IChargeable, IUpdatable{
Устройство может подключаться, заряжаться, обновляться
}
```
- Слабая связанность компонентов. Код зависит от интерфейса, а не от конкретной реализации.

### - Абстрактный класс используют для
- Общая функциональность семейства классов (класс Собака наследуется от класса Животное)
- Базовая реализация с возможностью переопределения.  Абстрактный класс может предоставить реализацию по умолчанию, которую наследник может заменить.
- Состояние и поля. Когда нужно хранить данные, общие для всех наследников. 

## -Философия интерфейсов и абстрактных классов
Интерфейс отвечает на вопрос "Что умеет делать?" (Летает - IFlyable)
Абстрактный класс отвечает на вопрос "Что это такое?" (Animal)
